# Claude 今日工作总结

## 适用场景

当用户希望你（Claude）把零散要点整理成**今日工作总结/日报**时使用。默认输出为**中文**，适合直接粘贴到飞书/企业微信/Slack/邮件。

## 快速开始（你需要从用户拿到什么）

优先把信息收集成下列字段；如果用户没提供，不要反复追问，按“未知/待确认”占位并给出你需要补充的最小问题清单。

- **日期**：默认今天（本地时区）
- **角色/项目**：例如「词汇挖掘」「EnLearning」
- **今日完成（Done）**：做了什么、产出了什么（代码/文档/PR/上线/结论）
- **关键结果（Outcomes）**：可量化指标、影响面、收益（如有）
- **进行中（In Progress）**：当前状态、下一步
- **风险/阻塞（Risks/Blockers）**：卡点、依赖谁、需要什么帮助
- **明日计划（Tomorrow）**：1-5 条，尽量可执行
- **链接/证据**：PR、Issue、文档（如有）

## 输出要求（默认规则）

- **结构清晰**：用标题 + 列表，信息密度高，避免空话
- **强调结果**：优先写“完成了什么/带来什么效果”，其次写“做了什么过程”
- **可复制粘贴**：不要用表情符号；不要用过长段落
- **不编造**：用户没给的数据/结论，明确标注“待确认”
- **敏感信息**：如涉及客户/隐私/密钥，自动泛化或移除

## 默认模板（标准版）

按以下格式直接输出（除非用户指定其他格式）：

```markdown
## 今日工作总结（YYYY-MM-DD）

### 今日完成
- [ ] ...

### 关键产出/影响
- ...

### 进行中
- ...

### 风险与阻塞
- ...

### 明日计划
- 1) ...
- 2) ...

### 需要协助（可选）
- ...
```

## 变体模板（按需选择）

### 精简版（3 段）

```markdown
## 今日进展（YYYY-MM-DD）
- Done：...
- Next：...
- Blockers：...
```

### 详细版（适合邮件/周会同步）

```markdown
## 今日工作总结（YYYY-MM-DD）

### 背景/目标（可选）
- ...

### 今日完成（按项目/模块分组）
#### 项目 A
- ...
#### 项目 B
- ...

### 数据与结论（可选）
- 指标：...
- 结论：...

### 风险与对策
- 风险：...；对策：...

### 明日计划（含验收口径）
- ...（验收：...）
```

## 生成流程（你作为 Claude 应如何做）

1. **解析用户要点**：把“动作/结果/证据/状态/阻塞/下一步”归类。
2. **补齐最小缺口**：如果缺关键信息，只提出不超过 3 个问题；否则直接生成并用“待确认”占位。
3. **输出总结**：默认用“标准版模板”；若用户说“很短/一句话/三点”，改用“精简版”。
4. **自检**：
   - 是否把“完成”与“计划”分开？
   - 是否有可交付物（链接/文件/PR/结论）？
   - 是否出现编造的指标/时间/结论？

## 示例

**输入要点：**
- 修复登录页在 Safari 的布局问题
- PR：#123 已合并，今晚发版
- 新增“今日生词”导出接口，完成接口联调（还差权限校验）
- 卡点：等后端确认字段 `word_level` 的枚举含义
- 明天：补权限校验、补单测、写一段 README 使用说明

**输出（标准版）：**

```markdown
## 今日工作总结（YYYY-MM-DD）

### 今日完成
- 修复登录页在 Safari 下的布局问题（已合并）
- 新增“今日生词”导出接口并完成联调（权限校验待补）

### 关键产出/影响
- PR：#123（已合并；发版计划：今晚）

### 进行中
- “今日生词”导出接口：权限校验与完善测试尚未完成

### 风险与阻塞
- 依赖后端确认 `word_level` 枚举含义（待确认）

### 明日计划
- 1) 补齐导出接口的权限校验
- 2) 补单元测试/关键路径测试
- 3) 补充 README 使用说明（含接口示例）
```
